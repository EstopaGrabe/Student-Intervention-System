def get_meta(student_data):
    data = []
    for col in student_data.columns:
        # Defining the role
        if col == 'passed':
            role = 'target'
        elif col == 'id':
            role = 'id'
        else:
            role = 'input'

        # Defining the level
        if student_data[col].dtype == np.int64:
            level = 'interval'
        elif student_data[col].dtype == np.float64:
            level = 'ordinal'

        # Initialize keep to True for all variables except for id
        keep = True
        if col == 'id':
            keep = False

        # Defining the data type 
        dtype = student_data[col].dtype

        # Creating a Dict that contains all the metadata for the variable
        col_dict = {
            'varname': col,
            'role'   : role,
            #'level' : level,
            'keep'   : keep,
            'dtype'  : dtype
        }
        data.append(col_dict)
    meta = pd.DataFrame(data, columns=['varname', 'role','keep','dtype'])
    meta.set_index('varname', inplace=True)
    return meta

Pré-processar Colunas de Atributo¶

Como voc pode ver, h muitas colunas no numricas que precisam ser convertidas! Muitas delas so simplesmente yes/no, por exemplo, a coluna internet. razovel converter essas variveis em valores (binrios) 1/0.

Outras colunas, como Mjob e Fjob, tm mais do que dois valores e so conhecidas como variveis categricas. A maneira recomendada de lidar com esse tipo de coluna criar uma quantidade de colunas proporcional aos possveis valores (por exemplo, Fjob_teacher, Fjob_other, Fjob_services, etc), e assinalar 1 para um deles e 0 para todos os outros.

Essas colunas geradas so por vezes chamadas de variveis postias (dummy variables), e ns iremos utilizar a funo pandas.get_dummies() para fazer essa converso. Execute a clula de cdigo abaixo para executar a rotina de pr-processamento discutida nesta seo.

def preprocess_features(X):
    ''' Pr-processa os dados dos estudantes e converte as variveis binrias no numricas em
        variveis binrias (0/1). Converte variveis categricas em variveis postias. '''
    
    # Inicialize nova sada DataFrame
    output = pd.DataFrame(index = X.index)

    # Observe os dados em cada coluna de atributos 
    for col, col_data in X.iteritems():
        
        # Se o tipo de dado for no numrico, substitua todos os valores yes/no por 1/0
        if col_data.dtype == object:
            col_data = col_data.replace(['yes', 'no'], [1, 0])

        # Se o tipo de dado for categrico, converta-o para uma varivel dummy
        if col_data.dtype == object:
            # Example: 'school' => 'school_GP' and 'school_MS'
            col_data = pd.get_dummies(col_data, prefix = col)  
        
        # Rena as colunas revisadas
        output = output.join(col_data)
    
    return output

Mquinas de vetores de suporte (SVM)

Introduo

As Mquinas de Vetores de Suporte (SVMs, do Ingls Support Vector Machines) constituem uma tcnica de aprendizado proveniente da Teoria do Aprendizado Estatstico, a qual est baseada no fato de que o erro da tcnica de aprendizagem junto aos dados de validao (erro de generalizao) limitado pelo erro de treinamento mais um termo que depende da dimenso. Os resultados da aplicao dessa tcnica so comparveis e muitas vezes superiores aos obtidos por outros algoritmos de aprendizado, como as Redes Neurais Artificiais (RNAs).

SVM implementa um mapeamento no-linear (executado por um produto interno kernel escolhido a priori) dos dados de entrada para um espao de caractersticas (feature space) de alta-dimenso, em que um hiperplano timo construdo para separar os dados linearmente em duas classes.

Quando os dados de treinamento so separveis, o hiperplano timo no espao de caractersticas aquele que apresenta a mxima margem de separao . Em suma, o algoritmo busca o hiperplano que tenha a distnica mxima entre os pontos prximos margem de separao conforme figura abaixo. Ao maximizar a margem, a classificao ou predio ter menos chance de overfitting.

K-Nearest Neighbors (KNeighbors)

Introduo

Utilizado tanto no aprendizado supervisionado quanto o no supervisionado, o mtodo K-Nearest Neighbors pode ser utilizado tanto para classificao para dados discretos e regresso para dados contnuos. Em sntese, o mtodo consiste em procurar os pontos prximos a um novo ponto e assim predizer/classificar. A distncia entre esses pontos pode ser determinada por diversos mtodos sendo o mtodo euclidiano o mais comum.

Por mais simples que seja, K-Nearest Neighbors apresenta timos resultados na classificao e regresso de problemas como monitoramento da camada de oznio e at mesmo a logstica de um sistema agricultor. Alm disso, sistemas de recomendao podem usar essa tcnica.

Vantagens

K-Nearest Neighbors possui algumas vantagens como o desempenho em grandes grupos de dados, simples de implementar alm de ser flexvel por atuar tanto com regresso quanto classificao. Pode-se ainda citar que conceitos complexos podem ser interpretados localmente por aproximao.

Desvantagens

As desvantagens desse mtodo so a determinao do parmetro k dos vizinhos prximos alm da funo distncia entre os pontos vizinhos quando a massa de dados muito grande.

rvores de Deciso

Introduo

rvores de deciso (DTs) so um mtodo de aprendizado de mquinas supervisionado para regresso e classificao. O objetivo criar um modelo que prediz/classifica o valor de uma varivel pelo aprendizado inferido sobre decises.

No caso da classificao, as rvores de deciso so representadas por instncias de "if-then" ou "se-ento" que resultam em um resultado classificatrio dos dados e, consequentemente, a uma deciso final.

Algumas observaes importantes sobre as DTs so que elas tendem a ter sobreajuste em grandes quantidades de atributos. Existem algoritmos para DTs como o ID3, C4.5, C5.0 e CART.

Vantagens

So simples de entender e representar, alm de serem versteis no sentido de se poder utilizar dados numricos e categricos. possvel ainda validar por mtodos estatsticos os modelos.

Desvantagens

Podem gerar rvores bastante complexas caso alguns cuidados no sejam tomados. Conjuntos de dados desbalanceados no tem um desempenho interessante caso no seja trabalhado esse desbalanceamento. Podem ser instveis em pequenas variaes dos dados e gerar resultados diferentes.

def train_classifier(clf, X_train, y_train):
    ''' Ajusta um classificador para os dados de treinamento. '''
    
    # Inicia o relgio, treina o classificador e, ento, para o relgio
    start = time()
    clf.fit(X_train, y_train)
    end = time()
    
    # Imprime os resultados
    print "O modelo foi treinado em {:.4f} segundos".format(end - start)
    return end - start


    
def predict_labels(clf, features, target):
    ''' Faz uma estimativa utilizando um classificador ajustado baseado na pontuao F1. '''
    
    # Inicia o relgio, faz estimativas e, ento, o relgio para
    start = time()
    y_pred = clf.predict(features)
    end = time()      
    teste_time = end - start
    # Imprime os resultados de retorno
    print "Pronto!\n As previses foram feitas em (secs): {:.4f}".format(end - start)
